¿Qué es TDD?

  • Técnica de Aprendizaje
    • Iterativa e Incremental
    • Constructivista
  • Basada en Feedback inmediato
  • Que recuerda todo lo aprendido
  • Y permite asegurarnos de no haber "desaprendido"
  • Incluye análisis, diseño, programacion y testing

¿Como se hace TDD?

  1. Escribir un test
    • Debe ser el más sencillo que se nos ocurra
    • Debe fallar al correrlo
  2. Correr todos los tests
    • Si hay errores, implementar lo mínimo necesario para que pasen y GOTO 2
  3. Reflexiono, ¿Se puede mejorar el código?
    • Si -> Refactorizar. GOTO 2
    • No -> GOTO 1

Ejemplo: 


							import unittest

							class CalculadoraTest(unittest.TestCase):
							
							    def test_cuando_sumo_1_mas_2_el_resultado_es_3(self):
							      calculadora = Calculadora();    
							      calculadora.suma(1,2)

							      self.assertEqual(3, calculadora.valor())

Tres reglas simples

"Solo puede escribir el código de producción (para implementar una funcionalidad) si ya ha escrito su respectivo código de prueba."
“ Solo puede escribir el código de prueba mínimo necesario que haga que el código de producción falle. ”
Solo puede escribir el código de producción necesario para hacer que éste pase su código de prueba.

Ventajas

Nos centramos en el Algorítmo
Mejora el diseño
Se asegura de que todo el código escrito está cubierto por una prueba

Desventajas

Tardamos más en terminar
Hay que cambiar muchos test si cambiamos código
Problema con la persistencia
